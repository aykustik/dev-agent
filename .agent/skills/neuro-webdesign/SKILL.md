# neuro-webdesign

Anwendbares Wissen über Neuropsychologie und Neurowissenschaft im Web. Direkt einsetzbar für Website-Konzeption, UI-Reviews, Landingpage-Feedback, Eye-Tracking-Simulation und A/B-Test-Hypothesen.

## 1. SYSTEM 1 & SYSTEM 2 — DAS FUNDAMENT

Das wichtigste Konzept für Webdesign aus Kahnemans Forschung:

### System 1 — Automatisch (schnell, emotional, unbewusst)
Reagiert in Millisekunden. Trifft ~95% aller Web-Entscheidungen.
- Emotionale Erstreaktion auf Design (gut/schlecht, vertrauenswürdig/unseriös — in 50ms)
- Visuelle Muster und Ankerpunkte
- Vertraute Designkonventionen (Navigation oben, Logo links)
- Gesichter, Bewegung, Farbe, Kontrast

**Design für System 1:**
- Vertraute Layoutmuster verwenden — Originalität kostet System-1-Vertrauen
- Emotionale visuelle Ankerpunkte setzen (Bild, Farbe, Gesicht)
- Klare visuelle Hierarchie damit System 1 sofort weiß "was ist wichtig"
- Erster Eindruck entscheidet: 50ms reichen für "vertraue ich dieser Website"

### System 2 — Bewusst (langsam, rational, anstrengend)
Reagiert nach Sekunden. Wird nur aktiviert wenn System 1 es zulässt.
- Liest Text, analysiert Angebote, vergleicht Preise
- Trifft bewusste Kaufentscheidungen
- Wird bei hoher kognitiver Last schnell müde

**Design für System 2:**
- Klare Informationshierarchie (H1 → H2 → Details)
- Kognitive Last reduzieren (Progressive Disclosure, weniger Optionen)
- Rationale Argumente strukturiert liefern (Bullet Points, Zahlen)
- Entscheidungshilfen: Empfehlungs-Badge, Vergleichstabellen, FAQs

### Das Zusammenspiel
**Reihenfolge ist entscheidend:** System 1 muss zuerst grünes Licht geben — erst dann liest System 2.
- Schlechtes Design → System 1 blockt → System 2 wird nie aktiviert → keine Conversion
- Gutes Design → System 1 sagt "ok, vertrauenswürdig" → System 2 liest und entscheidet
- Konsequenz: Ästhetik ist keine Dekoration — sie ist die Eingangstür zur rationalen Überzeugung

## 2. AUFMERKSAMKEIT & VISUELLE WAHRNEHMUNG

### Scan-Muster (Eye-Tracking-Forschung)

**F-Pattern** (textlastige Seiten: Blogs, Suchergebnisse, News)
- Nutzer lesen die erste Zeile vollständig (horizontal)
- Dann eine kürzere zweite horizontale Bewegung
- Dann scrollen sie vertikal links entlang
- → Wichtigstes oben links, Zweitwichtigstes in Zeile 2, Restinhalt links ausrichten

**Z-Pattern** (wenig Text, visuell: Startseiten, Landingpages)
- Oben links → oben rechts (Logo/Nav) → diagonal runter → unten links → unten rechts (CTA)
- → Primär-CTA unten rechts oder am Ende der Z-Diagonale platzieren

**Gutenberg-Diagramm**
- Seite in 4 Quadranten: oben links = starkes optisches Gewicht, unten rechts = Terminal-Zone
- → CTA in der Terminal-Zone (unten rechts) funktioniert gut nach langen Texten

### Pre-attentive Attributes (<200ms — vor bewusstem Sehen)
Das Gehirn filtert diese Attribute bevor System 2 überhaupt aktiv wird:
- **Farbe** — Ein rotes Element in grauer Umgebung wird sofort gesehen
- **Größe** — Größeres = wichtiger (funktioniert immer, überall)
- **Kontrast** — Hell auf dunkel, dunkel auf hell
- **Bewegung** — Animationen, Video-Autoplay (vorsichtig — erzeugt auch Ablenkung)
- **Orientierung** — Schräge Elemente in gerader Umgebung fallen auf
- **Form** — Ungewöhnliche Formen fallen auf (runde CTAs in eckigem Layout)
- **Umschlossene Flächen** — Rahmen, Cards, Hintergründe gruppieren Elemente

**Praktisch:** Ein einziger bunter CTA in einer monochromen Seite braucht keine Größe — die Farbe reicht.

### Gesichter & Blickrichtung
Menschliche Gesichter aktivieren System 1 sofort und lenken Aufmerksamkeit:
- Gesicht schaut zum CTA → Nutzer schaut dahin (Eye-Tracking belegt)
- Gesicht schaut zur Kamera → Nutzer schaut zurück ins Gesicht, ignoriert CTA
- Babies und Tiere haben denselben Effekt wie menschliche Gesichter

## 3. EMOTIONALES STORYTELLING

Storytelling aktiviert mehr Gehirnbereiche als reine Fakten und erzeugt stärkere emotionale Bindung.

### Warum Storytelling neurologisch wirkt
- **Neural Coupling**: Beim Hören/Lesen einer Geschichte synchronisiert sich das Gehirn des Hörers mit dem des Erzählers
- **Spiegelneuronen**: Erlebnisse in Geschichten werden vom Gehirn ähnlich verarbeitet wie eigene Erlebnisse
- **Oxytocin**: Empathische Geschichten triggern Oxytocin-Ausschüttung → Vertrauen und Großzügigkeit steigen

### Narrative Strukturen für Webdesign

**Problem → Konsequenz → Lösung** (für Landingpages)
1. Problem des Nutzers benennen (Empathie, System 1 sagt "das bin ich")
2. Konsequenz des ungelösten Problems zeigen (Schmerz vertiefen)
3. Produkt/Service als Lösung einführen

**Before → After → Bridge** (für Testimonials, Videos)
1. Zustand vor dem Produkt (Schmerz/Frustration)
2. Zustand nach dem Produkt (Transformation)
3. Das Produkt als Brücke dazwischen

**Held-Reise** (für "Über uns", Brand-Storytelling)
1. Ausgangssituation / Ursprung
2. Problem / Wendepunkt der den Brand-Gründung ausgelöst hat
3. Lösung / Mission / warum es dieses Unternehmen gibt

### Storytelling-Elemente im Design
- **Hero-Bild**: Zeigt nicht das Produkt — zeigt den Nutzer *nach* dem Produkt (Transformation)
- **Testimonials**: Immer als kurze Geschichte aufbauen (Situation → Problem → Lösung → Ergebnis)
- **Case Studies**: Ausführliche Version der Before-After-Bridge
- **Über uns**: Gründungsgeschichte die erklärt warum diese Lösung existiert
- **Produktbeschreibung**: Nicht "was es ist" sondern "wie dein Leben danach aussieht"

### Emotionale Textziele nach Kontext
Jede Seite sollte eine primäre Emotion ansteuern:

| Kontext | Ziel-Emotion | Design-Signal |
|---|---|---|
| Startseite | Neugier + Vertrauen | Klarer Hook, Social Proof, professionell |
| Landingpage | Dringlichkeit + Begehren | Loss Aversion, Transformation-Bild, CTA |
| Preisseite | Sicherheit + Rationalität | Garantie, FAQ, Vergleich, Decoy |
| Über uns | Sympathie + Vertrauen | Geschichte, Gesichter, Mission |
| Checkout | Sicherheit + Bestätigung | Trust-Signale, Progress, "Fast geschafft" |
| Fehlerseite | Humor + Orientierung | 404 mit Persönlichkeit, klare Weiterleitung |

## 4. KOGNITIVE LAST & VEREINFACHUNG

### Hick's Law
Je mehr Optionen, desto länger die Entscheidungszeit — und desto wahrscheinlicher Entscheidungslähmung.
- Navigation: max. 5-7 Punkte
- Pro Seite: ein primärer CTA — sekundäre CTAs deutlich kleiner/schwächer
- A/B-Test-Klassiker: Reduktion von 6 auf 3 Jam-Sorten = 10x mehr Käufe (Iyengar & Lepper 2000)

### Miller's Law
Kurzzeitgedächtnis fasst 7±2 Chunks:
- Navigationspunkte, Bullet-Listen (max. 5-7 Punkte)
- Formularfelder (in Schritten wenn >5)
- Feature-Listen auf Preisseiten

### Progressive Disclosure
Informationen häppchenweise:
- Accordion-FAQs statt langer FAQ-Seite
- Mehrstufige Formulare mit Progress-Indikator
- Checkout in 3-4 Schritte aufteilen

### Cognitive Fluency
Was sich leicht liest, wirkt vertrauenswürdiger und wahrer:
- Einfache Schriftarten für Fließtext
- WCAG AA: 4.5:1 Kontrast für Text
- Max. 3-4 Zeilen pro Absatz
- Weißraum = kognitive Pause = Vertrauen
- Bekannte Layouts nicht ohne Grund brechen

### Anchoring (Anker-Effekt)
Erster Wert beeinflusst alle folgenden Urteile:
- Preisseite: Teuerstes Paket zuerst → mittleres wirkt günstiger
- "Normalerweise 199€ — jetzt 99€" wirkt stärker als einfach "99€"
- Decoy Pricing: mittleres Paket neben "overpriced" Paket = mittleres wird bevorzugt

## 5. VERTRAUEN & GLAUBWÜRDIGKEIT

### Stanford Web Credibility Research (Fogg et al.)
Nutzer beurteilen Glaubwürdigkeit primär nach:
1. **Design-Qualität** — Schlechtes Design = kein Vertrauen, egal was der Inhalt sagt
2. **Informationsvollständigkeit** — Über uns, Impressum, echte Kontaktdaten
3. **Aktualität** — Veraltete Inhalte, © 2019 = sofortiger Vertrauensverlust
4. **Externe Links** — Verlinkung auf seriöse Quellen erhöht Glaubwürdigkeit
5. **Fehlerfreiheit** — Rechtschreibfehler, defekte Links = sofortiger Vertrauensverlust

### Social Proof
- Kundenzahlen **above the fold**: "Über 2.300 Kunden vertrauen uns"
- Testimonials: echtes Foto + voller Name + Unternehmen (nicht nur Initialen)
- Logo-Wall — auch wenn nur 3-4 bekannte Logos
- Case Studies > Testimonials (konkreter, glaubwürdiger)
- Video-Testimonials > Text-Testimonials

### Spezifische Trust-Elemente
- SSL + expliziter Hinweis beim Checkout
- Geld-zurück-Garantie prominent (reduziert wahrgenommenes Risiko)
- Datenschutz-Hinweis direkt beim Formular (nicht im Footer)
- Telefonnummer sichtbar (wird selten genutzt, aber Sichtbarkeit = Vertrauen)
- Echte Team-Fotos > Stock-Fotos
- Physische Adresse auffindbar

### Mere Exposure Effect (Zajonc 1968)
Wiederholte Exposition erhöht positive Bewertung:
- Retargeting-Ads wirken deshalb
- Konsistentes Branding über alle Touchpoints
- Wiederkehrende Nutzer konvertieren besser als Erstbesucher

## 6. VISUELLE HIERARCHIE & LAYOUT

### Gestalt-Prinzipien
**Nähe** — Nah = zusammengehörig
→ Formular-Label über dem Input, CTA neben dem Vorteil

**Ähnlichkeit** — Gleich aussehend = gleichwertig
→ Sekundär-CTAs einheitlich stylen, empfohlenes Paket hervorheben

**Fortsetzung** — Auge folgt Linien und Kurven
→ Pfeile und Flows leiten den Blick

**Abgeschlossenheit** — Gehirn vervollständigt unvollständige Formen
→ Teilweise sichtbare Karussell-Elemente signalisieren "mehr"

**Figur-Grund** — Objekte vom Hintergrund unterscheiden
→ CTA-Buttons brauchen Kontrast, Modals funktionieren dadurch

**Prägnanz** — Gehirn bevorzugt einfache, klare Formen
→ Einfache Icons > komplexe Icons

### Whitespace
Kein leerer Raum — aktives Design-Element:
- +20% Leseleistung durch mehr Whitespace (Studie)
- Lässt Premium-Produkte teurer wirken (Apple-Prinzip)
- Mikro: Zeilenabstand 1.4-1.6, Buchstabenabstand
- Makro: Abstände zwischen Sektionen = kognitive Pausen

### Typografische Hierarchie
- H1 → H2 → H3 → Fließtext: mind. 20% Größenunterschied
- Fett nur für das Wichtigste (wenn alles fett ist, ist nichts fett)
- Zeilenlänge: 45-75 Zeichen optimal (ca. 600-700px bei 16px)
- Dunkelgrau (#333-#444) auf Weiß angenehmer als Reinschwarz

## 7. FARB-PSYCHOLOGIE

### Grundassoziationen (DACH-Kulturraum)
| Farbe | Primär-Assoziation | System-1-Reaktion | Beste Einsatzbereiche |
|---|---|---|---|
| Blau | Vertrauen, Seriosität | Beruhigend, kompetent | B2B, Banken, Tech, Healthcare |
| Rot | Energie, Dringlichkeit | Alarmierend, aktivierend | Sale, Fehler-States, aggressive CTAs |
| Grün | Sicherheit, "Go" | Entspannend, bestätigend | Erfolgs-States, Ökologie, Bestätigung |
| Orange | Freundlichkeit, Zugänglichkeit | Einladend, warm | CTAs, Discount-Badges |
| Gelb | Aufmerksamkeit, Optimismus | Aufweckend | Highlights, Badges — nie für langen Text |
| Schwarz | Premium, Exklusivität | Distanziert, hochwertig | High-End, Fashion, Tech-Premium |
| Lila | Kreativität, Weisheit | Mystisch, kreativ | Beauty, kreative Branchen |
| Grau | Neutral, professionell | Unauffällig | Text, UI-Elemente, deaktivierte States |

### Farbe und Conversion
- CTA-Farbe muss **kontrastieren** — nicht harmonieren
- **Regel:** CTA-Farbe = die Farbe die sonst nirgends auf der Seite vorkommt
- Es gibt keine universell beste CTA-Farbe — Kontext entscheidet

### Accessibility
- WCAG AA: 4.5:1 für Normaltext, 3:1 für großen Text
- ~8% der Männer: Rot-Grün-Schwäche — nie nur Farbe als Indikator
- Immer zusätzlich Form/Icon/Text

## 8. EMOTIONALE TRIGGER

### Loss Aversion (Kahneman & Tversky)
Verluste wiegen psychologisch ~2x so schwer wie gleichwertige Gewinne.
- "Verpasse nicht..." > "Profitiere von..."
- Countdown-Timer (nur wenn echt)
- "Noch 3 Plätze" (nur wenn wahr)

### Reciprocity
Wer zuerst gibt, erhält wahrscheinlicher zurück.
- Lead Magnets, kostenlose Testphasen, wertvoller Content
- Das Geschenk muss als persönlich, unerwartet, wertvoll wahrgenommen werden

### Framing Effect
Wie gesagt wird ist wichtiger als was gesagt wird.
- "95% fettfrei" vs "5% Fett" — gleicher Inhalt, andere Wahrnehmung
- Positive Framing bei Benefits, Loss Aversion bei Dringlichkeit

### Commitment & Consistency
Wer einmal Ja sagte, sagt wahrscheinlicher wieder Ja.
- Mikro-Conversions vor der Haupt-Conversion
- Quiz/Konfigurator der Nutzer durch Schritte führt

### Endowment Effect
Was man besitzt (auch virtuell) gilt als wertvoller.
- Volle Testphase → Downgrade schmerzt
- "Dein Warenkorb" statt "Warenkorb"

## 9. EYE-TRACKING — SIMULATION & ANALYSE

Da echtes Eye-Tracking teuer ist, lässt sich mit Wissen und Methode eine fundierte Heatmap-Simulation erstellen.

### Virtuelle Heatmap erstellen
**Schritt 1 — Primärzone identifizieren** (sehr heiß)
- Oben links (Logo/erste Headline) — immer
- Erster CTA-Button
- Erstes Gesicht/Bild auf der Seite

**Schritt 2 — Sekundärzone** (warm)
- Headline und erste 2-3 Zeilen Text
- Navigation (beim ersten Besuch immer gescannt)
- Social Proof direkt unter dem Hero

**Schritt 3 — Kaltzonen** (kaum Aufmerksamkeit)
- Footer (außer Kontaktdaten die gesucht werden)
- Seitenleisten (rechte Seite wird oft ignoriert)
- Zweite und dritte Spalte bei Mehrspalten-Layouts

**Schritt 4 — Blickführungs-Check**
- Gibt es eine visuelle Linie vom ersten Ankerpunkt zum CTA?
- Führt das Layout den Blick oder verliert er sich?

### Häufige Eye-Tracking-Findings
- Nutzer ignorieren das, was wie eine Banner-Werbung aussieht (Banner Blindness)
- Bullet-Listen werden gescannt, nicht gelesen — erste 2-3 Wörter jedes Bullets sind entscheidend
- Preise werden in der Nähe von Referenzpunkten (Vergleichspreise) stärker wahrgenommen
- Horizontale Linien/Trennelemente stoppen den Scan-Flow

## 10. A/B-TEST-HYPOTHESEN NACH NEUROLOGISCHEN PRINZIPIEN

So leitet man aus neurowissenschaftlichen Erkenntnissen testbare Hypothesen ab:

### Hypothesen-Format
```
WENN [Änderung basierend auf Prinzip X]
DANN [erwartetes Verhalten]
WEIL [neurologische Begründung]
MESSGRÖSSE [Conversion Rate / CTR / Verweildauer / etc.]
```

### Beispiel-Hypothesen

**System 1 — Erstwahrnehmung:**
```
WENN wir das Hero-Bild von einem Produktbild auf ein Bild einer Person (die das Ergebnis des Produkts erlebt) ändern
DANN steigt die Verweildauer above the fold und die CTR auf den primären CTA
WEIL System 1 Gesichter und Transformationen schneller verarbeitet als abstrakte Produktdarstellungen
MESSGRÖSSE CTR auf Hero-CTA, Scroll-Tiefe
```

**Loss Aversion:**
```
WENN wir den CTA-Text von "Jetzt kaufen" auf "Meinen Platz sichern" ändern
DANN steigt die Conversion Rate
WEIL "sichern" Loss Aversion triggert (ich könnte meinen Platz verlieren)
MESSGRÖSSE Conversion Rate auf dieser Seite
```

**Hick's Law:**
```
WENN wir die Preispakete von 5 auf 3 reduzieren
DANN steigt die Gesamtconversion
WEIL weniger Optionen die Entscheidungslähmung reduzieren (Hick's Law)
MESSGRÖSSE Gesamtconversion auf Preisseite
```

**Social Proof Platzierung:**
```
WENN wir die Logo-Wall von unter dem Fold auf direkt unter dem Hero-CTA verschieben
DANN steigt die CTR auf den Hero-CTA
WEIL System 1 soziale Bewährtheit sofort vor der Entscheidung sehen muss
MESSGRÖSSE CTR auf Hero-CTA
```

**Cognitive Fluency:**
```
WENN wir Zeilenlänge auf max. 65 Zeichen begrenzen und Zeilenabstand auf 1.6 erhöhen
DANN steigt die Scroll-Tiefe und Verweildauer
WEIL bessere Lesbarkeit die kognitive Last senkt und Nutzer länger bleiben
MESSGRÖSSE Scroll-Tiefe, durchschnittliche Verweildauer
```

### Priorisierung von Tests
Nach potenziellem Impact:
1. Headline (größter einzelner Impact auf Conversion)
2. Hero-Bild (System-1-Erstreaktion)
3. CTA-Text und -Farbe
4. Social Proof Platzierung
5. Anzahl und Formulierung der Pakete/Optionen
6. Formular-Länge und Schrittanzahl

## 11. HIRNGERECHTE TEXT-ANFORDERUNGEN

Wenn Texte für maximale neurologische Wirkung benötigt werden, diese Parameter definieren:

### Template für Text-Anforderung
```
Kontext: [Headline / Subheadline / CTA / Body / Testimonial / FAQ-Antwort]
Ziel-Emotion: [Vertrauen / Dringlichkeit / Neugier / Sicherheit / Begehren / Empathie]
Zielgruppe: [wer liest das und was ist ihr aktueller Bewusstseinszustand]
Awareness-Level: [Unaware / Problem Aware / Solution Aware / Product Aware / Most Aware]
System-Ziel: [System 1 triggern (emotional, kurz) / System 2 überzeugen (rational, strukturiert)]
Länge: [max. X Wörter / Zeichen]
Ton: [Aykuts Stil / formal / conversational / dringend]
Einschränkungen: [Keine Klischees / Keywords einbauen / etc.]
```

### Beispiel-Anforderungen nach Kontext

**Hero-Headline (System 1, Neugier + Relevanz):**
```
Kontext: Hero-Headline
Ziel-Emotion: Neugier + "das ist für mich" Erkennung
Awareness-Level: Problem Aware
System-Ziel: System 1 triggern — muss in 50ms wirken
Länge: max. 8 Wörter
Anforderung: Benennt das Problem oder das Ergebnis, nicht das Produkt
```

**CTA-Text (System 1, Handlung):**
```
Kontext: Primärer CTA-Button
Ziel-Emotion: Motivation / leichte Dringlichkeit
System-Ziel: System 1 — sofortige Reaktion
Länge: 2-5 Wörter
Anforderung: Aktives Verb, nutzenorientiert, keine Passivformulierungen
```

## 12. MOBILE-SPEZIFIKA

- **Thumb Zone**: Daumen erreicht bequem unteren Bildschirmbereich — CTAs dort
- **Sticky CTA**: Fixierter Button unten hat höhere Tap-Rate
- **Touch Targets**: Min. 44×44px (Apple HIG), besser 48×48px
- **Above the Fold Mobile**: ca. 600-700px — Hook + Value Proposition + CTA rein
- **Formulare**: Keyboard-Typen setzen (tel, email, number), Autocomplete aktivieren
- **Ladezeit**: +1 Sekunde = ca. 7% Conversion-Verlust (Google/Deloitte)
- **Swipe-Signale**: Teilweise sichtbare Elemente = "hier gibt es mehr"

## 13. WEBSITE-ARCHITEKTUR NACH NEUROLOGISCHEN PRINZIPIEN

### Informationsarchitektur
- **Mentale Modelle respektieren**: Nutzer erwarten Bekanntes — Navigation, Suche, Logo links oben
- **Hierarchie vor Vollständigkeit**: Lieber weniger Punkte klar zeigen als alles auf einmal
- **Kognitive Pfade vorplanen**: Für jeden Nutzertyp (Erstkunde / Wiederkäufer / Researcher) einen klaren Pfad definieren

### Landingpage-Anatomie (neuropsychologisch optimiert)
1. **Hero** — System-1-Trigger: Emotion, Relevanz, Vertrauen in 3 Sekunden
2. **Problem** — Empathie aufbauen (Nutzer fühlt sich verstanden)
3. **Lösung** — Produkt als logische Antwort (nicht als Werbung)
4. **Wie es funktioniert** — System 2 beruhigen: klar, einfach, 3 Schritte
5. **Social Proof** — Bestätigung durch andere (System 1 + System 2)
6. **Features → Benefits** — Rational überzeugen
7. **Einwände eliminieren** — FAQ als Conversion-Retter
8. **Finaler CTA** — Mit Zusammenfassung des Nutzens + Risikominimierung

### Conversion-Architektur
- Jede Seite hat **genau einen** primären Conversion-Pfad
- Sekundäre Pfade sind deutlich schwächer gestaltet
- Zwischen Interesse und Handlung so wenig Klicks wie möglich

## 14. PRAKTISCHE REVIEW-CHECKLISTE

### System 1 Check
- [ ] Macht die Seite in 3 Sekunden klar worum es geht?
- [ ] Ist der erste visuelle Eindruck professionell und vertrauenswürdig?
- [ ] Gibt es einen klaren emotionalen Anker (Bild, Farbe, Gesicht)?
- [ ] Führt das Layout den Blick zum CTA?

### System 2 Check
- [ ] Ist die Informationshierarchie klar (H1 → H2 → Details)?
- [ ] Wie viele Entscheidungen muss der Nutzer treffen?
- [ ] Sind rationale Argumente strukturiert (Bullets, Zahlen, Vergleiche)?
- [ ] Sind die häufigsten Einwände beantwortet?

### Emotionales Storytelling
- [ ] Zeigt das Hero-Bild das Ergebnis/die Transformation?
- [ ] Gibt es eine erkennbare Narrativ-Struktur (Problem → Lösung)?
- [ ] Sprechen Testimonials von Transformation, nicht nur von Zufriedenheit?

### Vertrauen
- [ ] Social Proof above the fold sichtbar?
- [ ] Trust-Signale in der Nähe des CTAs?
- [ ] Wirkt das Design so professionell dass System 1 "ok" sagt?

### Conversion
- [ ] Ein primärer CTA pro Seite/Section?
- [ ] Nutzen klar bevor CTA kommt?
- [ ] Reibung vor der Conversion minimal?
- [ ] Loss Aversion oder Reziprozität eingebaut?

### Mobile
- [ ] CTA im Thumb-Bereich erreichbar?
- [ ] Touch-Targets ≥ 44px?
- [ ] Ladezeit < 3 Sekunden auf 4G?
